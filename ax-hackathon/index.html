근거 기반 컴플라이언스 린터 3종

규제기관이 이미 지목한 문제들 — 사용자 검증 누락, 드립 프라이싱, 의약품 오인 광고. 공식 문서·법조문·행정처분이라는 공개 근거를 결정론적 린트 규칙으로 변환해, 배포 전에 위반을 차단하는 Codex 플러그인 3종을 AI와 함께 혼자 만들고 골든셋으로 증명했습니다.

Overview

과제는 하나였습니다. 참여 기업의 실제 문제를 공개적으로 검증 가능한 자료만으로 입증하고, 그것을 해결하는 동작하는 Codex 플러그인을 제출하는 것.

대부분의 컴플라이언스 문제는 출시 이후에 잡힙니다 — 규제기관, 언론, 혹은 화난 사용자에 의해. 하지만 "무엇이 위반인가"의 근거는 이미 공개돼 있습니다. 공식 SDK 문서, 법조문, 행정처분 기록. 저는 그 근거를 CI에서 결정론적으로 도는 정적 분석 규칙으로 변환해, 같은 유형의 위반이 두 번 다시 배포되지 못하게 만들었습니다. 기업당 플러그인 하나, 세 기업, 세 개의 서로 다른 규제 도메인.

플러그인 3종

채널톡 · 메신저 SDK channel-boot-guard

근거: 공식 문서가 HMAC-SHA256 사용자 검증을 명시. 2023년 미동의 개인정보 이관 논란(KISA 통지)이 리스크를 실증.

핵심 primitive boot_objects() — 중괄호 균형 파싱으로 "파일 어딘가"가 아니라 "이 boot() 호출 안"에 memberHash가 있는지 판정. 임퍼소네이션·클라이언트 시크릿·profile PII·동의 전 boot 등 8개 규칙.

✓ 골든셋 15/15 · 유닛 28개 · PR diff 게이트

마이리얼트립 · OTA honest-ux-darkpattern

근거: 한국소비자원이 순차공개 가격책정으로 직접 지목(2026), 공정위는 사업자정보 미표시로 별도 제재. 전자상거래법 제21조의2.

핵심 primitive view_of() — "총액이 같은 화면 블록에 있는가"로 스코프를 좁혀, 페이지 하단 푸터 총액이 첫 화면 드립 위반을 지우지 못하게 함. 사업자번호 체크섬 포함 7개 규칙.

✓ 골든셋 12/12 · 유닛 21개 · 라이브 검증

메디테라피 · K-뷰티 cosmetic-ad-compliance

근거: 회사 자신이 의약품 오인 광고로 광고업무정지 3개월 처분(2025, 화장품법 제13조). 업계 행정처분 265건 중 광고가 177건으로 최다.

3층 엔진: PROHIBIT(금지 토큰) → CONTEXT(엑소좀❌ vs 식물 엑소좀✅ 문맥 판정) → SUBSTANTIATION(금지어 0개인 "재구매율 1위"까지 실증 요구) + 고정 변환표 리라이트.

✓ 골든셋 14/14 · 유닛 32개 · 조문 매핑

무엇이 달랐나

📎

모든 규칙에 출처가 있다

27개 규칙 전부가 조문 단위로 매핑됩니다 — 공식 문서 URL, 법조문, 행정처분 기록. 탐지 결과는 "느낌상 나쁘다"가 아니라 어떤 법·문서를 위반했는지와 그 링크를 함께 제시합니다.

⚙️

결정론, 추측이 아니라

순수 함수 엔진(Python stdlib만 사용): 같은 입력이면 같은 출력, CI에서 재현 가능. 휴리스틱 검사는 비차단 INFO로 강등해 오탐이 빌드를 막지 못하게 했습니다. 각 플러그인은 diff PR 게이트를 포함한 MCP 서버를 제공합니다.

🔬

스스로 반증하는 AI 활용

AI로 리서치 속도를 높이되, 모든 주장을 스스로 공격하는 적대적 검증을 통과시켰습니다. 버티지 못한 주장은 출시가 아니라 철회됐습니다 — UX 리서치에서 쓰는 가설-반증 루프와 같은 방식입니다.

진행 과정

01

근거 발굴

공식 문서·법조문·행정처분 — 누구나 재검증 가능한 공개 자료만 사용.

02

규칙 초안

위반 패턴마다 심각도·조문·출처 URL·수정안을 가진 규칙으로 변환.

03

적대적 검증

반례와 라이브 사이트로 내 규칙을 공격하고, 깨지는 주장은 철회.

04

구현

스코프 primitive + 규칙 엔진 + CLI + MCP 서버. 외부 의존성 없이.

05

골든셋 증명

실제 기업 문구와 규제 예외까지 라벨링한 bad/clean 쌍 — 41/41 통과.

스스로 철회한 주장들

가장 자랑스러운 건 린터가 잡아내는 것이 아니라, 제가 지운 것들입니다. 모든 철회 기록은 evidence 문서에 남겨두었습니다.

철회: "채널톡 profile 필드는 4개뿐" — setProperty로 custom field가 가능함을 확인. 규칙을 민감정보 denylist + 법조문 기반으로 재설계.
정정: "2023년 사고로 과징금" — 과징금 확정 사실 없음. "미동의 이관 패턴을 답습할 위험"으로만 표현.
삭제: CANCEL-FOOTERONLY 규칙 — 라이브 사이트에서 반증됨. 오탐을 만들기에 아예 출시하지 않음.
정정: 소비자원 수치는 6개 OTA 합산치로 회사 개별 비율이 아님 — "지목 후 문구를 추가했다"는 확인 가능한 사실만 유지.
