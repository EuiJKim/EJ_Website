저는 AI를 "써본" 사람이 아니라, AI와 함께 일하는 워크플로우 자체를 산출물로 가진 디자이너입니다. Claude Code는 단순한 챗봇이 아니라 파일과 코드베이스를 직접 읽고·쓰고·검증하는 도구이기에, 일회성 답변이 아닌 연속적 결과물을 만들어내는 파트너로 사용해왔습니다.

실제 산출물 · Real Artifacts

01 · 포트폴리오 웹사이트 + 다국어 i18n 시스템 euijkim.github.io/EJ_Website

메인 + 3개 케이스 스터디 + 회사별 지원 페이지까지 5개 이상의 라우트를 Claude Code로 직접 구축. ?lang=en URL 파라미터로 영어 모드 강제 진입, localStorage 동기화, data-en 속성 기반 1파일 i18n 시스템 설계.

02 · 자동화 PDF 생성 파이프라인 Puppeteer + pypdf

이력서·자기소개서·포트폴리오 PDF를 HTML 한 번 수정으로 동시 재생성. 1920×1080 캡처, A4 인쇄 최적화, 다중 PDF 병합, 페이지 사이즈 자동 정규화(1440→1920pt 스케일링)까지 한 스크립트로 처리.

03 · 회사별 맞춤 지원 페이지 시스템 /apply/[company]/

지원하는 회사마다 10분 안에 맞춤 자기소개서 페이지를 새로 만드는 템플릿화. 문토 · 페이타랩 · 인크로스 3개 운영 중. 인쇄 시 Quick Access 자동 숨김 등 print CSS 분리.

04 · 학술 분석 · 데이터 시각화 협업 ALLAW · KCI 2025

89명 사용자 응답 분석에서 코드·차트 생성, Think-Aloud 발화 클러스터링, 논문 초안 구조화 작업을 AI와 반복 협업. 판단은 사람이, 반복은 AI가 가져가는 흐름으로 학술 게재 수준까지 완성.

사용 원칙 · Working Principles

CORE RULE

AI가 판단을 대신하지 않는다. 사람이 판단에만 집중할 수 있도록 주변 마찰을 최대한 줄인다.

3-STEP WORKFLOW

① 가설 3~5개를 AI가 먼저 만든다 → 사람이 선택
② 산출물의 80%를 AI가 채운다 → 사람이 검토
③ 자가 점검을 AI에게 시킨다 → 누락 0%
