## 1. 평가자가 진짜 원하는 것 (핵심)

채용공고 + 과제 평가기준 + 애드테크 비즈니스 현실에서 도출:

| 층 | 평가자가 확인하려는 것 | 근거 |
|---|---|---|
| 1층 기본 | AI로 진짜 산출물을 빠르게 만드는가 | 공고 "AI 코딩툴 산출물", "생성형 AI 업무 효율화" |
| 2층 핵심 | AI를 **소비**하는가 **설계**하는가 | 직무명 "AI솔루션기획" |
| 3층 신뢰 | AI 결과를 **책임지는가** | 평가기준 "채택·폐기·판단", 애드테크=브랜드·광고비 |
| 차별 | **프론티어를 추적하고 맥락을 무기화**하는가 | AI 전담팀, Second Brain 명제 |

**한 문장 핵심**: "AI를 소비하지 않고 설계한다. 그리고 그 결과를 내가 책임진다."

## 2. 핵심 컨셉

> **"AI 정보 폭발을 내 무기로 바꾼다."**

AI를 "쓴다"가 아니라 **"AI 폭발 그 자체를 길들인다"** — 한 층 위의 메시지.
한눈에 "이 사람은 프론티어를 타고 논다 + 실제로 활용한다"가 박히도록 설계.

우선순위: **한눈에 읽히는 가독성 > 증거 밀도** (개념 나열 지양).

## 4. 영역별 내용 + 의도

### HERO — 3단 흐름 (페이지 주인공)

**① INPUT — 노이즈**
- 하루 수백 개 쏟아지는 AI 정보 (X · YouTube · GitHub)
- 시각: 흩어진 점/혼돈 느낌
- 메시지: "정보 과부하" 시대 정의

**② MY SECOND BRAIN — 정제 엔진** (차별점의 심장)
- Obsidian 볼트 + Claude Code 에이전트가 정제·요약·상호링크·질의 (LLM Wiki 패턴)
- "무엇을 내 맥락에 남기고 버릴지 큐레이션 기준은 내가" → 채택·폐기·책임 내장
- 시각: 깔끔한 노드 그래프 / 깔때기
- 하단 캡션: "맥락이 무기다"

**③ OUTPUT — 실제 결과**
- EEL 광고 콘텐츠 자동화 / 포트폴리오·지원 페이지 시스템 / ALLAW 분석
- 시각: 산출물 칩 3개
- 메시지: 노이즈가 실제 결과물로 전환됨

→ HERO 한 장면이 "활용"(input→output)과 "설계"(중앙 시스템)를 동시에 전달.

### 프론티어 흡수 타임라인 (얇은 띠)
- 시간 순 도구 흡수 궤적: 프롬프트 → Claude Code(harness) → agentic·subagent → MCP → Second Brain
- 의도: "트렌드를 계속 따라와 흡수한다"를 **명시적으로** 증명
- 주의: 단순 도구 나열이 아니라 "흡수 궤적"으로 보이게 (화살표/시간축)

### 한 줄 원칙
- "도구는 바뀌어도 내 맥락은 남는다." (Second Brain 영상의 플랫폼 독립성 명제)

## 5. 핵심 설계 결정

1. **HERO 3단 흐름이 페이지의 절반** — 가장 먼저, 가장 크게. 5초 가독성 우선
2. **Second Brain = 중앙 = 차별점** — "AI를 설계하는 사람"의 증거
3. **타임라인 띠로 "트렌드 추적" 분리 증명** — HERO(활용)와 다른 메시지축
4. **개념 나열 최소화** — harness·agentic·MCP 등은 타임라인 띠에만, HERO는 깔끔하게
5. **EEL은 과거 프로젝트 톤** — "현재 운영 중" 인상 배제
6. **A4 가로 1장** — 인쇄 깨짐 없이 한 화면
