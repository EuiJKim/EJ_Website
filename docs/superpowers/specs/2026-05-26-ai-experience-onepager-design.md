## 1. 배경

기존 `ai_experience.html`은 두 번 리디자인됨:
1. BUILD·OPERATE·GROW 3컬럼 버전 (정보 과다)
2. "한 사람이 운영하는 가구 사업, 실행은 AI 에이전트와 분담" Human↔AI 다이어그램 (역할 분담 상태만 보임)

문제: 인크로스 평가 기준은 **"AI를 얼마나 구조적으로 활용 — 어떤 맥락을 줬고, 무엇을 채택·폐기했고, 최종 판단을 어떻게 본인이 책임졌는가"**인데, 기존 버전은 "분담 상태"만 보이고 **채택·폐기·책임의 의사결정 루프**가 안 보임.

## 2. 핵심 컨셉

> **"AI를 도구가 아니라 워크플로우로 운영한다."**
> 평가 기준(맥락 → 채택·폐기 → 책임)을 페이지의 뼈대 자체로 삼는다.

전략: **하이브리드 C**
- **본체 (재명명)**: 이미 한 실제 작업을 프론티어 용어(agentic workflow, skill, MCP)로 재서술 — 안전·진실
- **프레시 1칸 (신규 적용)**: 반복 작업(`/apply/[회사]/` 지원 페이지 생성)을 Claude Code **스킬**로 자산화 → "최신 트렌드 추적 후 즉시 적용" 실증

lattice-log(AI 프론티어 큐레이션 플랫폼)는 **트렌드를 베껴오는 소스가 아니라, 내가 이미 하는 일을 최신 용어로 번역하는 지도**로 사용.

## 4. 영역별 내용 + 의도

### ① 운영 원칙 루프
- 4단계 가로 띠: 맥락 입력 → AI 다수 생성 → 채택·폐기 → 최종 책임
- 평가 기준 3요소(맥락/채택·폐기/책임)를 1:1로 시각화
- 하단 한 줄: "AI에게 판단을 맡기지 않는다. 반복을 맡기고 결정을 쥔다."

### ② 적용 사례 4칸 (2×2) — 모두 동일 4단 구조

| 칸 | 맥락 준 것 | AI가 한 것 | 채택·폐기 | 효과 |
|---|---|---|---|---|
| **EEL · 광고 콘텐츠** | 브랜드 톤·타깃 정의 | 카피·이미지 N버전 생성 | 브랜드 기준 미달분 폐기 | 콘텐츠 주기 주1회→1일 |
| **ALLAW · 분석** | 89명 응답·가설 | 데이터 분석·시각화·구조화 | 통념 가설 기각, 재설계 | KCI 등재후보지 게재 |
| **포트폴리오 시스템** | IA·디자인 토큰 | i18n·PDF 파이프라인 구축(agentic) | 수동 반복분 제거 | 1인이 다국어 사이트 운영 |
| **★ FRESH (2026.05)** | 반복 패턴(지원 페이지) | 슬래시 스킬로 자산화 | — | 신규 회사 페이지 20분→수분 |

- **핵심 설계**: 4칸이 똑같은 4단 틀 → "매번 같은 규율로 AI를 쓴다" = 구조성 증명
- **FRESH 칸만** 액센트 컬러 + "2026.05 적용" 뱃지로 최신성 강조

### ③ 도구 스택
- 프론티어 용어로 본인 도구 명명: Claude Code(주력) · agentic workflow · skills · MCP
- 한 줄: "프론티어 도구를 추적하고, 내 작업에 즉시 흡수한다."
